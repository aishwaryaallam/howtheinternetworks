HTTP and HTTPS Protocols

What is HTTP and how does it work?

HTTP also known as Hypertext Transfer Protocol is the foundation of any data exchange on the web. HTTP is an application-layer protocol that gives users a way to interact with web resources for transmitting hypermedia documents, such as HTML between clients and servers. HTTP clients generally use Transmission Control Protocol (TCP) connections to communicate with servers.

The step-by-step process of how HTTP works is:

  • Direct browser to URL: When a user wants to browse the internet, they type in a Uniform Recourse Locator (URL) in the browser. URL’s start with “http” which means that this is a signal to the browser that it needs to use HTTP to locate the document for that url.
  • Browser looks up IP: When a user types in a URL for example amazon.com the domain name map to the IP address, which is the true location of the domain’s computers. The browser uses a DNS resolver to map the domain to an IP address.
  • Browser sends HTTP request: Once the browser identifies the IP address of the computer that is hosting the requested URL it then sends an HTTP request. For example:

    GET /index.html HTTP/1.1

    Host: www.example.com

    The HTTP verb: “GET” is an action. There are other verbs for other actions such as “POST” used to submit form data. The next part specifies the path: “/index.html” because the computer stores the content of the entire website it is important to specify the path so that the browser knows which page to load. The final part of the first line “HTTP/1.1” specifies the protocol and the version of the protocol. Finally, the second line represents the domain of the requested URL.
  • Host Sends back HTTP Response: Once the host computer receives the HTT request, it then sends back a response with both the content and metadata about it.
  • The browser provides the response: The browser now has all the information it needs to provide the document that is being requested.
What is HTTPS?

HTTPS also known as Hypertext transfer protocol secure is the secure version of HTTP. The “S” in HTTPS stands for “Secure” meaning that all communications between your browser and the website are encrypted. HTTPS is used to protect and enables users to transmit sensitive data such as banking information, credit card numbers, etc. securely over the internet.
